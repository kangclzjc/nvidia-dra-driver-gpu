# --dry-run Mode Design for compute-domain-daemon

> Generated by Claude Code planner mode based on source code analysis

## Summary

Give compute-domain-daemon a `--dry-run` flag to skip IMEX hardware dependencies while preserving all K8s API operations (CDClique registration, DNS negotiation, readiness probes).

**Core principle:** Only 2 source files need changes (main.go + dnsnames.go) plus 2 template files. All changes guarded by `if flags.dryRun`, zero impact on normal mode.

## Files to Modify

### 1. cmd/compute-domain-daemon/main.go
- New `--dry-run` CLI flag (env: `DRY_RUN`, default: false)
- Skip CDI container edits check (still require COMPUTE_DOMAIN_UUID via env)
- Skip writeIMEXConfig()
- Skip ProcessManager creation and Watchdog goroutine
- New `dryRunIMEXDaemonUpdateLoop()` — consumes daemon info updates, does DNS mapping, skips IMEX process
- `check()` command returns success immediately in dry-run

### 2. cmd/compute-domain-daemon/dnsnames.go
- NewDNSNameManager() gets `dryRun bool` parameter
- /etc/hosts writes fallback to /tmp/hosts when not writable or in dry-run
- resolveHostsPath() auto-detects writable path

### 3. templates/compute-domain-daemon.tmpl.yaml
- Conditional --dry-run in command and probe commands via `{{- if .DryRun }}`

### 4. cmd/compute-domain-controller/daemonset.go
- New `DryRun bool` in DaemonSetTemplateData
- New `--dry-run-daemons` controller flag (env: `DRY_RUN_DAEMONS`)

## Files NOT Modified
- controller.go, cdclique.go, cdstatus.go, common.go, process.go, podmanager.go
- These are all pure K8s API operations, no hardware dependency

## Behavior Matrix

| Component | Normal | --dry-run |
|-----------|--------|-----------|
| CDClique registration | ✅ | ✅ |
| DNS negotiation | ✅ | ✅ (fallback /tmp/hosts) |
| Pod label update | ✅ | ✅ |
| IMEX config write | ✅ | ❌ skip |
| nvidia-imex process | ✅ | ❌ skip |
| Watchdog | ✅ | ❌ skip |
| check command | exec nvidia-imex-ctl | return success |

## Backward Compatibility

- DNSNameManager signature change is internal API only
- Template renders identically when DryRun=false
